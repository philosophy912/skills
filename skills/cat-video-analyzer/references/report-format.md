# 报告格式

每日报告输出到 `reports_dir/YYYY-MM-DD.md`。本文是格式的权威定义。

## 结构

```markdown
# 猫咪行为日报 · 2026-06-11

> 本报告由 cat-video-analyzer 自动生成。共处理 **120** 个视频文件，跳过已处理 **0** 个。

## 汇总

- 🍚 吃饭事件：**5** 次
- 🚽 上厕所事件：**3** 次

## 狸花猫（tabby）

| 行为 | 次数 | 总时长 |
|------|------|--------|
| 吃饭 | 3 | 22分15秒 |
| 上厕所 | 2 | 8分30秒 |

## 黑猫（black）

| 行为 | 次数 | 总时长 |
|------|------|--------|
| 吃饭 | 2 | 15分00秒 |
| 上厕所 | 1 | 4分12秒 |

## 时间线

| 时间 | 行为 | 猫咪 | 时长 | 证据 |
|------|------|------|------|------|
| 07:30:12-07:34:00 | 吃饭 | 狸花猫 | 3分48秒 | 狸花低头吃粮，鼻尖靠近金属碗 |
| 09:12:00-09:14:30 | 上厕所 | 黑猫 | 2分30秒 | 黑猫蹲伏在砂盆内刨沙 |
| ... | | | | |

---

*如需重新生成报告（不重新识别），运行 `python -m scripts.run --date 2026-06-11 --regenerate-report`。*
```

## 字段说明

- **汇总**：不分猫的总次数，用于一眼看全天活跃度。
- **每只猫小节**：按 `tabby → black → unknown` 固定顺序。即使某只猫当天无某类行为，
  也会显示 `0 次 / 0分00秒`，方便对比。
- **时间线**：所有事件按开始时间排列，含每条事件的识别证据摘要（截断 40 字）。
  证据是人工抽查识别质量的入口。

## 无活动时的报告

当天没识别到任何事件时，输出提示而非空表：

```markdown
## 本日无活动

今天没有识别到吃饭或上厕所事件。可能是猫咪休息日，也可能是红外夜视/画面遮挡
导致识别率下降，建议抽查原始视频。
```

## 重新生成

报告是 `raw/YYYY-MM-DD.jsonl` 的纯函数渲染。改报告格式后，跑：

```bash
python -m scripts.run report --date 2026-06-11
```

不消耗任何识别（agent 读图）开销，纯数据重聚合 + 重渲染，秒出。
