# 识别协议（Agent 必读）

本 skill 的逐帧识别**由你（agent）完成**，不是脚本调 API。本文是你识别每帧图片时
必须遵循的规则和输出格式。读图前先读这里。

## 你会拿到什么

`run extract` 命令会对每个视频抽帧，输出一个 JSON 数组，每个元素：

```json
{
  "frame_path": "/tmp/catva_xxx/frame_000001.jpg",
  "frame_ts": "2026-06-11T14:30:00+08:00",
  "context": "litter_box"
}
```

- `frame_path`：图片在磁盘上的路径。你用自己的读图能力（Read 工具 / 多模态）
  打开它。
- `frame_ts`：该帧的绝对时间（已由脚本算好 = 视频起始时间 + 帧偏移）。
- `context`：场景。**猫砂盆摄像头** → `litter_box`；**喂食机摄像头** → `feeder`。
  这个上下文对判断行为至关重要，不要忽略。

## 要识别什么

对每张图，判断两件事：

### 1. 画面里有哪些猫

家里两只猫：
- **狸花猫 `tabby`**：棕色或灰色**虎斑条纹**，额头有 M 形花纹，身上有明显的条纹/斑点。
- **黑猫 `black`**：全身黑色，无明显花纹。

如果画面里有猫但无法确定是哪只（比如只露了尾巴、画面太暗），填 `unknown`。
没有猫就返回空数组 `[]`。

### 2. 猫在做什么

根据 `context` 判断：

| context | 算 `toileting` | 算 `eating` | 否则 |
|---------|----------------|------------|------|
| `litter_box`（猫砂盆） | 刨沙、蹲伏在砂盆内、进出砂盆 | **不可能**（强行判 `idle`） | `idle`（路过/休息）或 `absent`（无猫） |
| `feeder`（喂食机） | **不可能**（强行判 `idle`） | 低头、鼻尖/嘴靠近食碗或出粮口、咀嚼 | `idle` 或 `absent` |

**关键约束**：猫砂盆画面里不会出现真正的进食，喂食机画面里不会出现真正的如厕。
如果你在 `litter_box` 画面里看到猫低头，那更可能是刨沙或嗅闻，不是吃饭。
这个约束在 `ingest` 阶段还会做代码兜底，但你在识别时就应正确归类。

## 输出格式

对每帧产出一个 JSON 对象（**纯 JSON，不要 markdown 代码块、不要前后文**）：

```json
{
  "frame_ts": "2026-06-11T14:30:00+08:00",
  "context": "litter_box",
  "cats": [
    {"identity": "black", "confidence": 0.9}
  ],
  "activity": {
    "type": "toileting",
    "evidence": "黑猫蹲伏在砂盆中央，前爪正在刨动猫砂"
  }
}
```

- `cats`：画面里每只猫一个元素。两只都在就两个。都没有就 `[]`。
- `activity.type`：`eating` / `toileting` / `idle` / `absent`。
  `absent` 仅当画面中完全没有猫。
- `activity.evidence`：**必须填**，用一句话引用画面细节。
  这不是走形式 —— 它逼你看图而非猜，也会写进最终报告的时间线供主人抽查。
  两只猫做不同的事时，在这里分别描述。

## 如何把结果写回

一个视频的所有帧都识别完后，把结果数组传给：

```bash
python -m scripts.run ingest \
  --video /path/to/clip.mp4 \
  --date 2026-06-11 \
  --results '[{"frame_ts":"...","context":"...","cats":[...],"activity":{...}}, ...]'
```

`ingest` 会：
1. 校验每个结果的字段合法性。
2. 应用路径级硬约束（猫砂盆目录的 `eating` → `idle`，反之亦然）。
3. 追加到 `raw/2026-06-11.jsonl`。
4. 把该视频标记为已处理（下次扫描跳过）。

## 常见陷阱

- **红外夜视画面**：猫可能只剩两个反光眼点。能判断行为就判断，判断不了就
  `idle`（有猫但状态不明）或保留 `unknown` identity。不要强行编造。
- **两只猫重叠/遮挡**：尽量分开判断；实在分不开，identity 填 `unknown`，
  evidence 注明"两只猫重叠难以区分"。
- **猫刚好吃完抬头**：这种过渡帧算 `idle`。只有明确的进食姿态才算 `eating`。
  宁可漏一帧（聚合时 30 秒间隔会补上），不要把 idle 混进 eating 污染统计。
